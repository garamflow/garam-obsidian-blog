# 아키텍처
## 1. Simple Database
- Primary
- 레디스 메인 프로세스 하나만 있다.
- 단 하나의 인스턴스만 존재해서 문제가 생기면 사용할 수 없다.
- 고가용성을 보장하지 못한다.
- 충분한 메모리와 서버 리소스를 혼자서 다 사용할 수 있다.

## 2. Ha Database
- 메인과 서브(복제)가 있다.
- 복제는 비동기식으로 동작한다.
- 메인에 들어오는 명령을 버퍼 형태로 서브에 전송하면서 복제한다.
- 어느정도의 가용성을 챙길 수 있다.
- 자동 복구를 지원하지 않기에 많이 사용하지 않는다.

- HA (고가용성)
	- 일종의 대비책이다.
	- 장애상황에 대해 자동 조치를 해주는 환경을 의미한다.
